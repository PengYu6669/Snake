# PRD: 拼形蛇 Shape Snake

## 1. 概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 拼形蛇（Shape Snake） |
| 定位 | 移动端轻量益智游戏 |
| 场景 | 黑客松展示，用户单次体验 2-3 分钟 |
| 核心理念 | 贪吃蛇骨架 × 拼图填充玩法 × 多关卡渐进难度 |
| 视觉风格 | 像素掌机外壳（GameBoy-like）+ 暗色调游戏画面 |

## 2. 目标用户 & 体验目标

- **用户画像**：黑客松评委、现场观众、移动端随手玩
- **体验目标**：打开即玩 → 15-30 秒理解规则 → 第1关3秒必过 → 第2关有挑战但可通过 → 后续关卡持续升级难度
- **设计原则**：零学习成本、渐进难度曲线、视觉反馈饱满、社交传播（失败海报）

## 3. 核心规则

### 3.1 关卡设计

| 维度 | 第1关·热身 | 第2关·进阶 |
|------|-----------|-----------|
| 网格 | 12×12 | 14×14 |
| 目标图形 | 十字形（5格），中心 (6,6) | 粗十字形（33格），横臂 rows 6-8 cols 4-10，纵臂 rows 4-10 cols 6-8 |
| 初始蛇长 | 5 | 5 |
| 初始蛇位 | 左侧横排 (y=6)，朝右 | 左侧横排 (y=9)，朝右 |
| 通关难度 | 自动向右3步即通关 | 需策略填充33格，可通关 |
| 步数限制 | 99（宽松） | 400 |
| 时间限制 | 无 | 120秒 |
| 资源块 | 无 | 28个金色资源块（沿边框分布，总质量5+28=33=目标格数） |
| 撞墙/撞自己 | 仅震动，不死亡 | 直接 Game Over |
| 通关行为 | 自动跳转第2关 | 全屏粒子庆祝 |

### 3.2 操作定义

| 操作 | 效果 | 步数 |
|------|------|------|
| 转向 | 改变蛇头朝向 | - |
| 移动 | 蛇在朝向上自动前进1格（350ms/步） | +1 |
| 吐方块 | 从蛇尾放下一个静态方块，蛇长度 -1 | +1 |

### 3.3 通关条件

1. 目标图形每格都被占用（蛇身或吐块）
2. 目标区域外无占用格
3. 占用总格数 = 目标格数

### 3.4 回收机制

- 蛇头经过已吐出的方块时自动回收，方块消失，蛇长 +1
- 蛇头经过金色资源块时吃掉，蛇长 +1（不消耗步数资源）
- 资源可循环利用，避免死局

### 3.5 失败条件（第2关）

- 步数达到上限（400步）
- 倒计时归零（120秒）
- 撞墙或撞到自己

## 4. 操作方式

| 平台 | 转向 | 吐块 | 重置 |
|------|------|------|------|
| 掌机方向键 | 十字键点击 | A键（红色） | B键（蓝色） |
| 移动端 | 滑动屏幕（四向滑动） | 双击屏幕 / A键 | B键 |
| 桌面端 | 方向键 / WASD | 空格 / Enter | B键 |

## 5. 视觉设计

### 5.1 掌机外壳
- 暗紫色机身（#2a2332），圆角边框，内凹屏幕区域
- 顶部像素装饰条（粉色圆点）
- 扬声器格栅（底部右侧斜纹）
- 四角装饰螺丝
- 屏幕区域玻璃反光效果
- 品牌文字 "SHAPE-SNAKE SYSTEM"
- 响应式缩放（适配小屏和矮屏）

### 5.2 游戏画面（Canvas 渲染）
- 暗色网格背景（12×12 / 14×14）
- 第1关：冷色调网格（灰蓝）
- 第2关：炼狱色调（红色脉动）
- 蛇身 HSL 渐变色（头亮红 → 尾暗红）
- 蛇头圆角矩形 + 眼睛随方向移动瞳孔
- 蛇头运动残影拖尾（最近8帧轨迹）
- 目标区域脉动虚线边框（绿色）
- 金色资源块：旋转菱形 + 金色辉光脉动
- 吐块：绿色方块 + 弹出缩放动画
- 填充目标格：✓ 标记
- 粒子效果：填充、收集、庆祝等多种粒子系统
- 撞墙/撞自己：屏幕震动（+ 手机震动）
- 第2关倒计时 < 10s：屏幕边缘红色脉动
- 通关：全屏闪白 + 持续粒子喷发
- 游戏未开始：半透明遮罩 + 操作提示

## 6. 音效系统（Web Audio API）

### 6.1 音效清单

| 音效 | 触发时机 | 类型 |
|------|---------|------|
| click | 蛇移动 | square 700Hz |
| glissando | 转向 | triangle 200→650Hz |
| success | 填充目标格 | 三连音 (523, 659, 784) |
| bump | 撞墙震动 | sawtooth + 噪音 |
| winFanfare | 通关 | 七音阶上行 |
| loseSound | 失败 | sawtooth 下滑 + 噪音 |
| collect | 吃金色资源 | 双升调 (880→1108Hz) |

### 6.2 背景音乐
- **第1关 BGM**：135bpm，欢快风格（square主旋律 + triangle低音，lowpass 1400Hz）
- **第2关 BGM2**：150bpm，紧张风格（A小调，square旋律 + 切分低音 + sawtooth铺底，lowpass 800Hz）→ 后续关卡可替换

### 6.3 iOS 兼容
- AudioContext 在用户手势中同步创建（iOS 要求）
- 每3秒保活检查（suspend→resume）

## 7. 失败海报（Canvas 生成）

- 第2关失败时生成 375×812（@2x）海报
- 深色渐变背景 + 网格纹理 + 噪点
- 顶部「拼形蛇 · 第二关」「😵 挑战失败」
- 大字「别笑」白色+黑色描边 + 红色辉光
- 副文案「「你也过不了第二关」」
- 进度环（已填充 / 总目标）
- 数据卡（已填充 / 差一点 / 总目标）
- 支持 navigator.share 分享，回退下载

## 8. 技术实现

### 8.1 技术栈
- **渲染**：Canvas 2D + requestAnimationFrame
- **音频**：Web Audio API（SoundSynth 类）
- **构建**：Vite 8
- **依赖**：无第三方运行时依赖，单文件实现
- **包管理**：npm（仅 devDependency: vite）

### 8.2 响应式
- 掌机 max-width: 420px，屏幕自适应
- Canvas 逻辑尺寸通过 DPR 缩放保证清晰度
- 三档断点适配（>700px / ≤700px / ≤400px）
- 移动端保护：user-select: none、overscroll-behavior: none、touch-action: manipulation
- 禁止长按菜单（contextmenu 阻止）

### 8.3 关键机制
- 连续移动系统（RAF 循环 + 350ms 间隔计时）
- 双关卡配置化（LEVELS 数组）
- 步数/时间双维度失败检测
- 资源块 Map（O(1) 查找）
- 吐块竞态保护（lastSpitStep 守卫）
- 游戏未开始不计时（gameStarted 标志）
