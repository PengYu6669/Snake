# PRD: 拼形蛇 (Shape Snake)

## 1. 概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 拼形蛇（Shape Snake） |
| 定位 | 移动端轻量益智游戏 |
| 场景 | 黑客松展示，用户单次体验 2-3 分钟 |
| 核心理念 | 贪吃蛇骨架 × 拼图填充玩法 × 交互式引导 |
| 视觉风格 | 像素掌机外壳（GameBoy-like）+ 暗色调游戏画面 |
| 技术形态 | 单 HTML 文件，零外部依赖，纯前端 |

---

## 2. 用户流程

```
打开页面
  └→ 教程 Overlay（默认全屏显示）
      ├→ 点击"跳过教程" → 加载第 1 关
      ├→ 逐帧观看动画 → 在蛇尾等待时双击吐块
      │    → 3 次交互后自动播放完成 → 点击"开始挑战" → 加载第 1 关
      └→ 通关后点击"再学一次"重置教程
            └→ 加载第 1 关 → 通关 → 1.5 秒自动切换到第 2 关
                  ├→ 通关 → 结算面板
                  │    ├→ 通关第 2-4 关 → HTML 结算面板 + 称号
                  │    │    ├→ "下一关"（最后一关显示"再玩一次"）
                  │    │    └→ "重来"（重玩当前关）
                  │    └→ 通关第 5 关 → 全屏粒子 + Canvas 生成庆祝海报 + 结算面板
                  ├→ 失败（第 1 关） → 全屏 GAME OVER + 分级失败文案 → 重试
                  ├→ 失败（第 2-4 关） → 全屏 GAME OVER + 分级失败文案 → 重试
                  └→ 失败（第 5 关） → 生成分享海报 + 分级失败文案 → 重试/分享
```

## 3. 核心规则

### 3.1 操作定义

| 操作 | 方式 | 效果 | 步数 |
|------|------|------|------|
| 转向 | 方向键 / 滑动 / DPad | 改变蛇头朝向，触发一次移动 | +1 |
| 自动前进 | 350ms 间隔，RAF 驱动 | 蛇在朝向上自动前进 1 格 | +1 |
| 吐方块 | A 键 / 双击 / 空格/Enter | 从蛇尾放下静态绿色方块，蛇长 -1 | +1 |

**批量吐块**：蛇尾在目标格内时，自动检测尾部连续在目标格内的段，一次性全部吐出。粒子数 = 吐出数 × 5，播放 success 音效。

### 3.2 碰撞规则

| 场景 | 第 1 关（热身） | 第 2-5 关 |
|------|----------------|-----------|
| 撞墙 | 仅震动，不死亡 | 直接 Game Over |
| 撞自己 | 仅震动，不死亡 | 直接 Game Over |

### 3.3 通关条件（双约束）

1. 目标图形每格都被占用（蛇身经过 或 吐块落在目标格）
2. 蛇身和吐块不占用任何非目标格（总占据格数 = 总目标格数）

### 3.4 资源与回收

- **金色资源块**：蛇头经过时吃掉，蛇长 +1，触发金色粒子 + 震动
- **吐块回收**：蛇头经过已吐出的方块时自动回收，方块消失
- 资源可循环利用，避免死局
- 资源位置由 `generateResources()` 运行时随机生成（基于曼哈顿距离，取距离目标 1-3 格的候选位）

### 3.5 失败条件

- 步数达到关卡上限（各关不同）
- 倒计时归零（第 2-5 关各有时间限制）
- 第 2-5 关撞墙或撞到自己

### 3.6 失败消息分级

基于 `getRemainingCells()` 计算的未填充目标格数，显示对应文案：

| 剩余格数 | 文案 |
|----------|------|
| 1-3 | 仅差 X 格！再来一次必过 (filled/total) |
| 4-10 | 差 X 格，已经很近了 (filled/total) |
| 11-30 | 还差 X 格，继续挑战 (filled/total) |
| 30+ | 还剩 X 格，路还很长 (filled/total) |

## 5. 交互式教程

### 5.1 设计
- 5×5 网格，十字目标（5 格）
- 13 帧硬编码动画剧本，无坐标计算
- 蛇长 3，依次演示：蛇移动 → 蛇尾在目标格等待吐块 → 蛇身经过自动填充

### 5.2 交互点
| 时机 | 操作 | 教学目的 |
|------|------|---------|
| 帧 160（蛇尾在目标格#1） | 双击吐块 | 尾部吐块填目标 |
| 帧 200（蛇尾在目标格#3） | 双击吐块 | 重复练习吐块 |
| 帧 220（蛇身经过目标格#2） | 自动触发 | 蛇身经过也是填充 |

### 5.3 视觉元素
- 隐藏吐块目标指示器（蛇尾闪烁虚线框）
- 进度圆点（5 个对应 5 个十字目标格）：未完成/活跃/已完成
- 文案提示变化："双击屏幕吐方块" / "蛇身也是方块！"
- 通关后粒子爆发 + "蛇身+吐块=填满目标"

## 6. 视觉设计

### 6.1 掌机外壳（CSS）
- 暗紫色机身 `#2a2332`，圆角边框，内凹屏幕区域

- 响应式三档适配（全高 >700px / 矮屏 ≤700px / 小屏 ≤400px）

### 6.2 游戏画面（Canvas 渲染）
- 暗色网格背景，棋盘格明暗交替
- 第 1 关冷色调（灰蓝），第 2-5 关地狱风（红色脉冲 + 边框红外发光）
- 蛇身 HSL 渐变色（头亮红 `#ff6b6b` → 尾暗红），蛇头圆角矩形 + 眼睛随方向移动瞳孔
- 蛇头运动残影拖尾（最近 8 帧轨迹，红色渐变透明）
- 目标区域脉动虚线边框 + 浅绿底色呼吸
- 金色资源块：旋转菱形 + 金色辉光脉动
- 吐块：绿色方块 + 弹射缩放动画 + 绿色辉光
- 已填充目标格：绿色半透明底 + ✓ 标记
- 粒子系统：收集/填充/庆祝多种粒子（颜色、扩散、重力参数可配）
- 碰撞反馈：屏幕震屏 + 设备振动
- 倒计时 < 10s：屏幕边缘红色脉动光晕
- 通关：全屏闪白 + 持续彩色粒子喷发
- 游戏未开始：半透明遮罩 + "滑动屏幕开始" + 操作提示

### 6.3 结算/称号系统
- 每关通关后显示结算面板（第 1-3 关为简单 HTML div，第 4 关同 div）

## 7. 音效系统（Web Audio API，SoundSynth 类）

| 音效 | 触发 | 合成方式 |
|------|------|---------|
| click | 蛇移动 1 格 | square 700Hz, 40ms |
| glissando | 转向 | triangle 200→650Hz 滑音 |
| success | 填充目标格 | 三连音 523/659/784Hz |
| bump | 碰撞震动 | sawtooth 120→60Hz + 白噪声 |
| winFanfare | 通关 | square 上行琶音 C5→E5→G5→C6，每音 120ms |
| loseSound | 失败 | sawtooth 400→100Hz 下滑 + square 80Hz 咚 |
| collect | 吃金色资源 | 双升调 880→1108Hz |

**背景音乐**：
- 第 1 关 BGM：135bpm，C 大调，square 主旋律 + triangle 低音，lowpass 1400Hz
- 第 2-3 关 BGM2 1.0x：150bpm，A 小调，紧张切分节奏 + sawtooth 铺底，lowpass 800Hz
- 第 4-5 关 BGM2 1.2x：180bpm，A 小调，lowpass 600Hz（升级版）

**兼容**：AudioContext 用户手势创建 + 3 秒保活轮询

## 8. 失败海报系统

- 第 5 关失败时 Canvas 生成 375×500 分享海报（@2x DPR）
- 深色渐变背景 + 网格纹理 + 随机噪点
- 顶部："拼形蛇 · 第五关" / "😵 挑战失败"
- 大字 "别笑"（白色 + 黑色描边 + 红色辉光）
- 副文案 "「你也过不了第五关」"
- 进度环（已填充 / 总目标百分比）
- 数据卡片：已填充 | 差一点 | 总目标
- 分级失败文案显示在海报下方
- 支持 `navigator.share`（含文件）→ 回退下载

## 10. 技术架构

### 10.1 技术栈
| 层 | 技术 |
|----|------|
| 渲染 | Canvas 2D + requestAnimationFrame (60fps) |
| 音频 | Web Audio API（SoundSynth 自包含合成器） |
| 构建 | Vite 8（仅开发用，产物为单文件） |
| 依赖 | 零第三方运行时依赖 |
| 兼容 | 现代移动端浏览器，iOS AudioContext 适配 |

### 10.2 项目结构
```
snake/
├── index.html       # 主游戏文件（~2370 行，单体 HTML+CSS+JS）
├── PRD.md           # 产品需求文档
├── 功能测试.md      # 功能测试用例文档
├── test.html        # 原型参考（历史文件，不参与构建）
├── package.json
├── package-lock.json
├── vite.config.js
└── .gitignore
```

### 10.3 核心系统模块
| 模块 | 职责 | 关键实现 |
|------|------|---------|
| 关卡系统 | 关卡数据、TARGET Set、资源 Map | `LEVELS` 数组（5 关），`loadLevel()` 初始化 |
| 资源生成 | 运行时随机生成资源位置 | `generateResources()` 曼哈顿距离 + Fisher-Yates 洗牌 |
| 移动循环 | 驱动蛇前进 | RAF + 350ms 间隔守卫，`tryMove()` |
| 方向保护 | 禁止 180° 掉头 | `OPPOSITE` 映射表，`DIR_STR` 字符串化比较 |
| 特效引擎 | 粒子 / 脉冲 / 震屏 / 拖尾 | `fx` 全局对象，每帧 `updateEffects()` |
| 音效引擎 | 合成音效 + BGM 排程 | `SoundSynth` 类，`_scheduleBGM()` 递归 setTimeout |
| 胜利系统 | 双约束判定 + 庆祝动画 | `checkWin()` 遍历 TARGET + all-occupied Set |
| 失败系统 | 三条件检测 + 分级文案 + 振动 | `handleFail()` → `showLoseScreen()` / `showPoster()` |
| 结算系统 | 通关后显示结算面板 | `showSettleScreen()` 显示 HTML 结算 div + 称号 |
| 称号系统 | 基于进度 5 级称号 | `getTitle(pct)` → 🐣🐍🐲👑🔥 |
| 教程系统 | 帧剧本动画 + 等待交互 | IIFE 自执行，13 帧 SCRIPT 数组，`doSpit()` 蛇尾吐块 |
| 海报系统 | 失败分享图片生成 | `generatePoster()` Canvas 2D 绘制 375×500（第 5 关） |
| 庆祝海报 | 通关庆祝图片生成 | `generatePoster(won=true)` Canvas 2D 绘制 375×812（第 5 关） |

### 10.4 关键数据结构
- `TARGET = new Set()` — `"x,y"` 字符串键，O(1) 查找
- `spitBlocks = new Map()` — 吐块位置 → `true` 或 `{spitProgress}`
- `resourceBlocks = new Map()` — 资源位置 → `true`
- `fx` — 特效状态：particles[], pulses[], trail[], shake, flashAlpha
- `SCRIPT[]` — 教程帧剧本：`{frame, snake[], hint, wait?, fill?, complete?}`
- `DIR_STR` / `OPPOSITE` — 方向字符串化与反向映射
