# PRD: 拼形蛇 (Shape Snake)

## 1. 概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 拼形蛇（Shape Snake） |
| 定位 | 移动端轻量益智游戏 |
| 场景 | 黑客松展示，用户单次体验 2-3 分钟 |
| 核心理念 | 贪吃蛇骨架 × 拼图填充玩法 × 交互式引导 |
| 视觉风格 | 像素掌机外壳（GameBoy-like）+ 暗色调游戏画面 |
| 技术形态 | 单 HTML 文件，零外部依赖，纯前端 |

---

## 2. 用户流程

```
打开页面
  └→ 教程 Overlay（默认全屏显示）
      ├→ 点击"跳过教程" → 加载第 1 关
      ├→ 逐帧观看动画 → 在蛇尾等待时双击吐块
      │    → 3 次交互后自动播放完成 → 点击"开始挑战" → 加载第 1 关
      └→ 通关后点击"再学一次"重置教程
            └→ 加载第 1 关
                  ├→ 通关 → 1.5 秒自动切换到第 2 关
                  │    └→ 通关 → 全屏粒子庆祝 → 显示通关面板
                  │          ├→ "下一关"（当前为最后一关时显示"再玩一次"）
                  │          └→ "重来"（重玩当前关）
                  └→ 失败（第 1 关） → 全屏 GAME OVER + 分级失败文案 → 重试
                  └→ 失败（第 2 关） → 生成分享海报 + 分级失败文案 → 重试/分享
```

## 3. 核心规则

### 3.1 操作定义

| 操作 | 方式 | 效果 | 步数 |
|------|------|------|------|
| 转向 | 方向键 / 滑动 / DPad | 改变蛇头朝向，触发一次移动 | +1 |
| 自动前进 | 350ms 间隔，RAF 驱动 | 蛇在朝向上自动前进 1 格 | +1 |
| 吐方块 | A 键 / 双击 / 空格/Enter | 从蛇尾放下静态绿色方块，蛇长 -1 | +1 |

### 3.2 碰撞规则

| 场景 | 第 1 关 | 第 2 关 |
|------|---------|---------|
| 撞墙 | 仅震动，不死亡 | 直接 Game Over |
| 撞自己 | 仅震动，不死亡 | 直接 Game Over |

### 3.3 通关条件（双约束）

1. 目标图形每格都被占用（蛇身经过 或 吐块落在目标格）
2. 蛇身和吐块不占用任何非目标格（总占据格数 = 总目标格数）

### 3.4 资源与回收

- **金色资源块**：蛇头经过时吃掉，蛇长 +1，触发金色粒子
- **吐块回收**：蛇头经过已吐出的方块时自动回收，方块消失
- 资源可循环利用，避免死局
- 第 2 关配置 35 个资源块，包围目标区分布

### 3.5 失败条件

- 步数达到关卡上限（第 1 关 99 步 / 第 2 关 400 步）
- 倒计时归零（第 2 关 120 秒）
- 第 2 关撞墙或撞到自己

### 3.6 失败消息分级

基于 `getRemainingCells()` 计算的未填充目标格数，显示对应文案：

| 剩余格数 | 文案 |
|----------|------|
| 1-3 | 仅差 X 格！再来一次必过 (filled/total) |
| 4-10 | 差 X 格，已经很近了 (filled/total) |
| 11-30 | 还差 X 格，继续挑战 (filled/total) |
| 30+ | 还剩 X 格，路还很长 (filled/total) |

## 4. 关卡设计

| 特性 | 第 1 关·热身 | 第 2 关·进阶 |
|------|-------------|-------------|
| 网格 | 12×12 | 14×14 |
| 目标形状 | 小十字（5 格），中心 (6,6) | 粗十字（33 格），横臂 rows 6-8 cols 4-10，纵臂 rows 4-10 cols 6-8 |
| 初始蛇位 | `[{4,6},{5,6},{6,6},{7,6},{8,6}]`，朝右 | `[{4,9},{5,9},{6,9},{7,9},{8,9}]`，朝右 |
| 初始蛇长 | 5 | 5 |
| 步数上限 | 99 | 400 |
| 时间限制 | 无 | 120 秒 |
| 资源块 | 0 | 35（目标区外围 1-2 格环形分布） |
| 通关方式 | 自动跳转第 2 关 | 庆祝动画 → 通关面板 |
| 通关难度 | 操作 3 步即通关 | 需策略规划 33 格填充 |

## 5. 交互式教程

### 5.1 设计
- 5×5 网格，十字目标（5 格）
- 13 帧硬编码动画剧本，无坐标计算
- 蛇长 3，依次演示：蛇移动 → 蛇尾在目标格等待吐块 → 蛇身经过自动填充

### 5.2 交互点
| 时机 | 操作 | 教学目的 |
|------|------|---------|
| 帧 160（蛇尾在目标格#1） | 双击吐块 | 尾部吐块填目标 |
| 帧 200（蛇尾在目标格#3） | 双击吐块 | 重复练习吐块 |
| 帧 220（蛇身经过目标格#2） | 自动触发 | 蛇身经过也是填充 |

### 5.3 视觉元素
- 隐藏吐块目标指示器（蛇尾闪烁虚线框）
- 进度圆点（5 个对应 5 个十字目标格）：未完成/活跃/已完成
- 文案提示变化："双击屏幕吐方块" / "蛇身也是方块！"
- 通关后粒子爆发 + "蛇身+吐块=填满目标"

## 6. 视觉设计

### 6.1 掌机外壳（CSS）
- 暗紫色机身 `#2a2332`，圆角边框，内凹屏幕区域
- 顶部 5 颗像素装饰指示灯（粉色，3 秒呼吸闪烁）
- 品牌文字 "SHAPE-SNAKE SYSTEM"
- 四角装饰螺丝 + 底部右侧扬声器格栅
- 屏幕区域玻璃反光 CSS 伪元素
- DPad 十字键 + A/B 按钮（红/蓝）+ Start/Select 长条按钮
- 响应式三档适配（全高 >700px / 矮屏 ≤700px / 小屏 ≤400px）

### 6.2 游戏画面（Canvas 渲染）
- 暗色网格背景，棋盘格明暗交替
- 第 1 关冷色调（灰蓝），第 2 关地狱风（红色脉冲 + 边框红外发光）
- 蛇身 HSL 渐变色（头亮红 `#ff6b6b` → 尾暗红），蛇头圆角矩形 + 眼睛随方向移动瞳孔
- 蛇头运动残影拖尾（最近 8 帧轨迹，红色渐变透明）
- 目标区域脉动虚线边框 + 浅绿底色呼吸
- 金色资源块：旋转菱形 + 金色辉光脉动
- 吐块：绿色方块 + 弹射缩放动画 + 绿色辉光
- 已填充目标格：绿色半透明底 + ✓ 标记
- 粒子系统：收集/填充/庆祝多种粒子（颜色、扩散、重力参数可配）
- 碰撞反馈：屏幕震屏 + 设备振动
- 倒计时 < 10s：屏幕边缘红色脉动光晕
- 通关：全屏闪白 + 持续彩色粒子喷发
- 游戏未开始：半透明遮罩 + "滑动屏幕开始" + 操作提示

## 7. 音效系统（Web Audio API，SoundSynth 类）

| 音效 | 触发 | 合成方式 |
|------|------|---------|
| click | 蛇移动 1 格 | square 700Hz, 40ms |
| glissando | 转向 | triangle 200→650Hz 滑音 |
| success | 填充目标格 | 三连音 523/659/784Hz |
| bump | 碰撞震动 | sawtooth 120→60Hz + 白噪声 |
| winFanfare | 通关 | 七音阶上行 262→1047Hz |
| loseSound | 失败 | sawtooth 350→50Hz 下滑 + 噪声 |
| collect | 吃金色资源 | 双升调 880→1108Hz |

**背景音乐**：
- 第 1 关 BGM：135bpm，C 大调，square 主旋律 + triangle 低音，lowpass 1400Hz
- 第 2 关 BGM2：150bpm，A 小调，紧张切分节奏 + sawtooth 铺底，lowpass 800Hz

**兼容**：AudioContext 用户手势创建 + 3 秒保活轮询

## 8. 失败海报系统

- 第 2 关失败时 Canvas 生成 375×500 分享海报（@2x DPR）
- 深色渐变背景 + 网格纹理 + 随机噪点
- 顶部："拼形蛇 · 第二关" / "😵 挑战失败"
- 大字 "别笑"（白色 + 黑色描边 + 红色辉光）
- 副文案 "「你也过不了第二关」"
- 进度环（已填充 / 总目标百分比）
- 数据卡片：已填充 | 差一点 | 总目标
- 分级失败文案显示在海报下方
- 支持 `navigator.share`（含文件）→ 回退下载

## 9. 技术架构

### 9.1 技术栈
| 层 | 技术 |
|----|------|
| 渲染 | Canvas 2D + requestAnimationFrame (60fps) |
| 音频 | Web Audio API（SoundSynth 自包含合成器） |
| 构建 | Vite 8（仅开发用，产物为单文件） |
| 依赖 | 零第三方运行时依赖 |
| 兼容 | 现代移动端浏览器，iOS AudioContext 适配 |

### 9.2 项目结构
```
snake/
├── index.html       # 主游戏文件（~2140 行，单体 HTML+CSS+JS）
├── PRD.md           # 产品需求文档
├── test.html        # 原型参考（历史文件，不参与构建）
├── package.json
├── package-lock.json
├── vite.config.js
└── .gitignore
```

### 9.3 核心系统模块
| 模块 | 职责 | 关键实现 |
|------|------|---------|
| 关卡系统 | 关卡数据、TARGET Set、资源 Map | `LEVELS` 数组，`loadLevel()` 初始化 |
| 移动循环 | 驱动蛇前进 | RAF + 350ms 间隔守卫，`tryMove()` |
| 特效引擎 | 粒子 / 脉冲 / 震屏 / 拖尾 | `fx` 全局对象，每帧 `updateEffects()` |
| 音效引擎 | 合成音效 + BGM 排程 | `SoundSynth` 类，`_scheduleBGM()` 递归 setTimeout |
| 胜利系统 | 双约束判定 + 庆祝动画 | `checkWin()` 遍历 TARGET + all-occupied Set |
| 失败系统 | 三条件检测 + 分级文案 | `handleFail()` → `showLoseScreen()` / `showPoster()` |
| 教程系统 | 帧剧本动画 + 等待交互 | IIFE 自执行，13 帧 SCRIPT 数组，`doSpit()` 蛇尾吐块 |
| 海报系统 | 失败分享图片生成 | `generatePoster()` Canvas 2D 绘制 375×500 |

### 9.4 关键数据结构
- `TARGET = new Set()` — `"x,y"` 字符串键，O(1) 查找
- `spitBlocks = new Map()` — 吐块位置 → `true` 或 `{spitProgress}`
- `resourceBlocks = new Map()` — 资源位置 → `true`
- `fx` — 特效状态：particles[], pulses[], trail[], shake, flashAlpha
- `SCRIPT[]` — 教程帧剧本：`{frame, snake[], hint, wait?, fill?, complete?}`

## 10. 已实现 vs 待改进

### ✅ 已完成
- 像素掌机外壳 + 响应式布局
- 双关卡系统（12×12 热身 + 14×14 进阶）
- 双约束通关判定（全部覆盖 + 零额外占用）
- 蛇尾吐块机制 + 回收
- 金色资源收集系统
- 交互式帧动画教程（3 次用户交互）
- 第 2 关 120 秒倒计时 + 紧急警告
- 失败分级文案系统
- Canvas 生成分享海报 + 分享/下载
- Web Audio 合成音效 + 双 BGM
- 粒子/脉冲/震屏/闪白特效
- 键盘 + 触屏 + DPad 三模输入

### 🔄 待改进
- 关卡数量不足（仅 2 关），缺少难度曲线
- 目标图形单一（仅有十字形）
- 无关卡选择 / 跳关功能
- 无暂停 / 悔步
- 无存档（LocalStorage），刷新丢失进度
- 无评分 / 排行榜 / 成就
- 只有失败海报，缺少胜利分享图
- 无 PWA / 离线支持
- 海报 `roundRect` 兼容性依赖
- 教程 EventListener 重复绑定隐患
