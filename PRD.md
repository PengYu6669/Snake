# PRD: 拼形蛇 (Shape Snake)

## 1. 概述

| 项目 | 内容 |
|------|------|
| 产品名称 | 拼形蛇（Shape Snake） |
| 定位 | 移动端轻量益智互动 |
| 场景 | 抖音互动空间上传，用户单次体验 2-5 分钟 |
| 核心理念 | 贪吃蛇骨架 × 拼图填充玩法 × 三种挑战模式 |
| 视觉风格 | 像素掌机外壳（GameBoy-like）+ 暗色调互动画面 |
| 技术形态 | 单 HTML 文件，零外部依赖，纯前端离线运行 |

---

## 2. 用户流程

```
打开页面
  └→ 主菜单（模式选择）
      ├→ 🐍 闯关模式 → 显示"开始挑战"按钮 → 8 关逐步挑战
      │    ├→ 通关 → 结算面板（星星评价 + 称号 + 得分）
      │    │    ├→ "下一关" / "重来" / "返回主页"
      │    │    └→ 第 8 关通关 → 全屏粒子庆祝 + 海报
      │    └→ 失败 → 结算面板（分级失败文案）→ "重来" / "返回主页"
      ├→ ⚡ 极限竞速 → 显示"开始挑战"按钮 → 一口气通关全部 8 关
      │    ├→ 关间自动过渡（无 start overlay）
      │    └→ 通关/失败 → 结算面板（总用时 + 最佳纪录对比）
      ├→ ♾ 无尽挑战 → 显示"开始挑战"按钮 → 波次推进，难度递增
      │    ├→ 每波完成 → 下一波（目标+2，步数预算微增）
      │    └→ 步数耗尽/碰撞 → 结算面板（波数 + 填充格 + 总步数）
      └→ 📖 新手教程 → 5×5 网格动画演示 → "开始挑战" / "跳过教程"
```

## 4. 核心规则

### 4.1 操作定义

| 操作 | 方式 | 效果 | 步数 |
|------|------|------|------|
| 转向 | 方向键 / 滑动 / DPad | 改变蛇头朝向，触发一次移动 | +1 |
| 自动前进 | 380ms 间隔，RAF 驱动 | 蛇在朝向上自动前进 1 格 | +1 |
| 吐方块 | A 键 / 双击 / 空格/Enter | 从蛇尾放下静态绿色方块，蛇长 -1 | +1 |

**批量吐块**：蛇尾在目标格内时，自动检测尾部连续在目标格内的段，一次性全部吐出（至少保留 2 节蛇身）。粒子数 = 吐出数 × 5，播放 success 音效。

### 4.2 碰撞规则

| 场景 | 第 1 关（热身） | 第 2-8 关 / 竞速 / 无尽 |
|------|----------------|------------------------|
| 撞墙 | 仅震动，不死亡 | 直接挑战结束 |
| 撞自己 | 仅震动，不死亡 | 直接挑战结束 |

### 4.3 通关条件（双约束）

1. 目标图形每格都被占用（蛇身经过 或 吐块落在目标格）
2. 蛇身和吐块不占用任何非目标格（总占据格数 = 总目标格数）

### 4.4 资源与回收

- **金色资源块**：蛇头经过时吃掉，蛇长 +1，触发金色粒子 + 震动
- **吐块回收**：蛇头经过已吐出的方块时自动回收，方块消失
- 资源可循环利用，避免死局
- 资源位置由 `generateResources()` 运行时随机生成（基于曼哈顿距离，取距离目标 1-3 格的候选位）

### 4.5 失败条件

- 步数达到关卡上限（各关不同）
- 倒计时归零（第 2-8 关各有时间限制）
- 第 2-8 关撞墙或撞到自己
- 无尽模式：当前波步数预算耗尽

### 4.6 失败消息分级

基于 `getRemainingCells()` 计算的未填充目标格数，显示对应文案：

| 剩余格数 | 文案 |
|----------|------|
| 1-3 | 仅差 X 格！再来一次必过 (filled/total) |
| 4-10 | 差 X 格，已经很近了 (filled/total) |
| 11-30 | 还差 X 格，继续挑战 (filled/total) |
| 30+ | 还剩 X 格，路还很长 (filled/total) |

## 5. 三种挑战模式

### 5.1 闯关模式

标准 8 关逐步挑战，每关独立结算。

- 通关后显示结算面板（星星、称号、得分）
- 支持选关（点击关卡名打开选关面板）
- 进度通过 localStorage 持久化

### 5.2 极限竞速

一口气通关全部 8 关，计总用时。

- 开始后计时器持续运行（不暂停）
- 关间自动过渡，不显示 start overlay
- 结算显示总用时，与历史最佳对比
- 最佳用时存储在 localStorage

### 5.3 无尽挑战

波次推进，难度线性递增，直到失败。

- 12×12 固定网格
- 每波目标数递增（+2/波，上限 12）
- 每波资源块数 = 目标数（1:1 比例）
- 步数预算：基础 50 + 每波 3 + 每目标 6
- 剩余步数结转到下一波
- 蛇身因资源块不断变长，操作空间逐渐压缩
- 步数到 70% 变黄、90% 变红、80% 触发心跳音效
- 结算显示波数、填充格数、总步数
- 个人排行榜（top 5），按波数 > 填充格 > 步数排序
- 结算面板提供"📊 排行榜"入口

**无尽模式难度曲线：**

| 波次 | 目标数 | 资源数 | 步数预算 | 每目标步数 |
|------|--------|--------|----------|-----------|
| 0 | 4 | 4 | 74 | 18.5 |
| 1 | 6 | 6 | 89 | 14.8 |
| 2 | 8 | 8 | 104 | 13.0 |
| 3 | 10 | 10 | 119 | 11.9 |
| 4 | 12(封顶) | 12 | 134 | 11.2 |
| 6 | 12 | 12 | 140 | 11.7 |

### 5.4 特殊事件波系统

第 1 波起，每波开始时有概率触发一个特殊事件（随机 5 种）。触发后显示 3 秒事件横幅（banner），游戏暂停，结束后恢复。

| 图标 | 名称 | 效果 | 视觉/颜色 |
|------|------|------|-----------|
| 🌀 | 反向波 | 方向键反转，持续整波 | 四角旋转紫色箭头 `#a855f7` |
| ⚡ | 加速波 | 蛇移动间隔 380ms→250ms | 白色速度线划过屏幕 |
| 🎁 | 礼物波 | 地图生成 3 颗星星，吃掉 +10 步数 | 金色旋转星星 `#ffd700` |
| 💥 | 收缩波 | 每 12 秒外圈收缩 1 层死亡区；蛇头进入即死；死亡区内的目标格被移除 | 死亡区红格 + 岩浆发光边框 + 倒计时脉动 `#ef4444` |
| 🪞 | 镜像波 | 以屏幕中心对称生成镜像蛇，方向与主蛇相反；吐块时镜像蛇同步吐块，填充效率翻倍 | 蓝色半透明蛇 + 中心十字对称虚线 `#38bdf8` |

**设计意图**：
- **反向波/加速波**：操作干扰，增加操作难度
- **礼物波**：正向奖励，步数续航
- **收缩波**：空间压缩，迫使用户积极填充而非消极游走，与"生存"主题高度契合
- **镜像波**：双倍填充效率，正向反馈；镜像蛇存活时用户需注意走位避免撞墙

每个事件结束时自动清理状态，下一波重新随机选取。

## 6. 分数与星级系统

### 6.1 得分计算

**通关：**
```
得分 = 覆盖分(0-100) + 步数奖励(0-50) + 时间奖励(0-50)
```

**失败：**
```
得分 = 覆盖分(0-100)
```

- 覆盖分 = (已填充 / 总目标) × 100
- 步数奖励 = (maxSteps - steps) / maxSteps × 50（仅通关时计算）
- 时间奖励 = (timeLimit - timeUsed) / timeLimit × 50（仅通关时计算）

### 6.2 星级评价

| 星级 | 条件 |
|------|------|
| ★ | 通关（任意完成度） |
| ★★ | 100% 填充 |
| ★★★ | 100% 填充 + 步数 ≤ 45% 上限 + 时间 ≤ 50% 上限 |

### 6.3 称号系统（基于完成度）

| 进度 | 称号 | 图标 |
|------|------|------|
| ≤30% | 菜蛇 | 🐣 |
| ≤60% | 挣扎蛇 | 🐍 |
| ≤85% | 毒蛇 | 🐲 |
| ≤99% | 蛇王 | 👑 |
| 100% | 炼狱幸存者 | 🔥 |

### 6.4 结算面板

通关时显示：
- 星星动画（3 颗大星星依次弹出亮起）
- 称号徽章
- 新纪录提示（🏆 新纪录！）
- 完成度 / 步数 / 用时 / 得分
- 操作按钮

### 6.5 选关面板

卡片式网格布局（3 列），每张卡片显示：
- 关卡编号
- 3 颗星星（获得的亮金色，未获得的暗灰）
- 最佳分数
- 未解锁的显示锁图标
- 顶部总星数汇总（⭐ X / 24）

## 8. 交互式教程

### 8.1 设计
- 5×5 网格，十字目标（5 格）
- 13 帧硬编码动画剧本，无坐标计算
- 蛇长 3，依次演示：蛇移动 → 蛇尾在目标格等待吐块 → 蛇身经过自动填充

| 帧 220（蛇身经过目标格#2） | 自动触发 | 蛇身经过也是填充 |

### 8.3 视觉元素
- 隐藏吐块目标指示器（蛇尾闪烁虚线框）
- 进度圆点（5 个对应 5 个十字目标格）：未完成/活跃/已完成
- 文案提示变化："双击屏幕吐方块" / "蛇身也是方块！"
- 通关后粒子爆发 + "蛇身+吐块=填满目标"

## 10. 音效系统（Web Audio API，SoundSynth 类）

| 音效 | 触发 | 合成方式 |
|------|------|---------|
| click | 蛇移动 1 格 | square 700Hz, 40ms |
| glissando | 转向 | triangle 200→650Hz 滑音 |
| success | 填充目标格 | 三连音 523/659/784Hz |
| bump | 碰撞震动 | sawtooth 120→60Hz + 白噪声 |
| winFanfare | 通关 | square 上行琶音 C5→E5→G5→C6，每音 120ms |
| loseSound | 失败 | sawtooth 400→100Hz 下滑 + square 80Hz 咚 |
| collect | 吃金色资源 | 双升调 880→1108Hz |
| heartbeat | 濒死（步数/时间 80%） | 低频脉冲 |

**背景音乐**：
- 第 1 关 BGM：135bpm，C 大调，square 主旋律 + triangle 低音，lowpass 1400Hz
- 第 2-4 关 BGM2 1.0x：150bpm，A 小调，紧张切分节奏 + sawtooth 铺底，lowpass 800Hz
- 第 5-8 关 BGM2 1.2x：180bpm，A 小调，lowpass 600Hz（升级版）
- 无尽模式 BGM2：同第 2-4 关

**兼容**：AudioContext 用户手势创建 + 3 秒保活轮询

## 13. 技术架构

### 13.1 技术栈
| 层 | 技术 |
|----|------|

| 兼容 | 现代移动端浏览器，iOS AudioContext 适配 |

### 13.2 项目结构
```
snake/
├── index.html       # 主互动文件（~4000 行，单体 HTML+CSS+JS）
├── PRD.md           # 产品需求文档
├── 互动空间创作.md   # 抖音互动空间创作规范
```

### 13.3 核心系统模块
| 模块 | 职责 | 关键实现 |
|------|------|---------|
| 主菜单 | 模式选择 + 最佳记录展示 | `showMainMenu()`，三个模式卡片 |
| 关卡系统 | 关卡数据、TARGET Set、资源 Map | `LEVELS` 数组（8 关），`loadLevel()` 初始化 |
| 资源生成 | 运行时随机生成资源位置 | `generateResources()` / `generateEndlessResources()` |
| 移动循环 | 驱动蛇前进 | RAF + 380ms 间隔守卫，`tryMove()` |
| 方向保护 | 禁止 180° 掉头 | `OPPOSITE` 映射表，`DIR_STR` 字符串化比较 |
| 特效引擎 | 粒子 / 脉冲 / 震屏 / 拖尾 | `fx` 全局对象，每帧 `updateEffects()` |
| 音效引擎 | 合成音效 + BGM 排程 | `SoundSynth` 类，`_scheduleBGM()` 递归 setTimeout |
| 胜利系统 | 双约束判定 + 庆祝动画 | `checkWin()` 遍历 TARGET + all-occupied Set |
| 失败系统 | 三条件检测 + 分级文案 + 振动 | `handleFail()` → `showSettleScreen()` |
| 结算系统 | 通关后显示结算面板 + 星星动画 | `showSettleScreen()` + `calculateStars()` |
| 评分系统 | 得分计算 + 星级评价 + 持久化 | `calculateScore()` / `saveLevelScore()` |
| 选关系统 | 卡片式选关 + 星星展示 | `updateLevelSelectUI()` 动态生成卡片 |
| 竞速模式 | 累计计时 + 关间过渡 | `initSpeedRun()` / `speedRunNextLevel()` |
| 无尽模式 | 波次推进 + 难度缩放 + 资源生成 | `initEndlessMode()` / `endlessNextWave()` |
| 特殊事件系统 | 5 种随机事件波（反向/加速/礼物/收缩/镜像） | `currentEvent` + `EVENT_INFO` + `showEventBanner()` |
| 收缩波 | 外圈死亡区逐步收缩 + 安全区倒计时 | `shrinkBorder`/`shrinkTimer` + gameLoop 每帧递减 |
| 镜像波 | 屏幕中心对称镜像蛇 + 双倍填充 | `mirrorSnake[]`/`mirrorAlive` + tryMove 中跟随移动 |
| 称号系统 | 基于进度 5 级称号 | `getTitle(pct)` → 🐣🐍🐲👑🔥 |
| 教程系统 | 帧剧本动画 + 等待交互 | IIFE 自执行，13 帧 SCRIPT 数组 |
| 海报系统 | 分享图片生成 | `generatePoster()` Canvas 2D 绘制 |

### 13.4 关键数据结构
- `TARGET = new Set()` — `"x,y"` 字符串键，O(1) 查找
- `spitBlocks = new Map()` — 吐块位置 → `{spitProgress}` 或 `true`
- `resourceBlocks = new Map()` — 资源位置 → `true`
- `fx` — 特效状态：particles[], pulses[], trail[], shake, flashAlpha
- `SCRIPT[]` — 教程帧剧本：`{frame, snake[], hint, wait?, fill?, complete?}`
- `DIR_STR` / `OPPOSITE` — 方向字符串化与反向映射
