# 贪拉蛇 · 互动空间适配优化清单

对照《互动空间创作 Skill》逐项自检，以下为待优化项。

---

## P0 · 必须修复

### 1. 错误兜底缺失

**现状**：游戏运行中如果报错（如 Canvas 初始化失败、JS 异常），用户直接看到白屏或浏览器默认报错。

**要求**：必须添加全局错误兜底（参考 Skill 的"参考错误兜底"模板）。

**改动**：在 `</script>` 前追加 `window.addEventListener('load', ...)` 包裹 `initApp()`，并提供一个 fallback div 展示"哎呀，出错了，请重启试试~"。

---

### 2. localStorage 存档缺失

**现状**：完全没有持久化存储，关闭页面后进度丢失。

**要求**：互动空间作品应默认支持最高分/进度存档，`localStorage` key 必须带业务前缀。

**改动**：
- 使用 `snake_unlock` 前缀保存已解锁的最高关卡
- 每次通关时将当前关卡写入 localStorage
- `loadLevel` 时从 localStorage 读取解锁状态
- 选关界面中，已解锁关卡可点击，未解锁关卡置灰

---

## P1 · 建议优化

### 3. 弹层结构未遵循推荐模板

**现状**：
- 弹层使用 `.overlay` 类，而非推荐的 `.screen`/`.modal-card` 结构
- 存在多个 `.overlay.show` 同时激活的可能性（例如胜利弹层 + 结算弹层并存）
- 缺少 `aria-*` 无障碍属性

**要求**：自绘弹层应采用类名切换的覆盖层结构，任一时刻只允许一个主弹层处于激活态。

**改动**：
- 将 `.overlay` 重构为 `.screen` 容器 + `.modal-card` 卡片结构
- 新增 `hideAllScreens()` 统一管理弹层状态
- 显示任一弹层前先关闭其他弹层
- 添加 `aria-hidden` / `aria-modal` 属性

### 4. 暂停状态使用 Canvas 绘制而非 DOM 弹层

**现状**：暂停时在 Canvas 上绘制"已暂停·点击继续"文字，而非 DOM 弹层。

**要求**：暂停类弹层应有明确的恢复、重开等按钮，关闭后状态可恢复。

**改动**：将暂停界面改为 DOM 覆盖层，包含"继续"、"重来"两个按钮，与现有的 Canvas pause 绘制二选一。

---

## P2 · 锦上添花

### 5. 分享海报下载文件名

**现状**：`a.download = 'shapesnake.png'` 为英文文件名。

**建议**：改为 `贪拉蛇_挑战结果.png` 更符合中文互动空间场景。

### 6. 屏幕方向锁定

**现状**：未锁定屏幕方向。竖屏掌机设计在横屏设备上显示异常。

**建议**：添加 CSS `@media (orientation: landscape)` 提示用户旋转设备，或通过 Screen Orientation API 锁定竖屏。

### 7. 教程中 dblclick 在移动端兼容性

**现状**：教程 Canvas 使用 `dblclick` 监听双击触发吐块教学，但移动端双击会触发缩放。

**建议**：改为 `touchend` 计时检测（短时间两次触摸视为双击），与主游戏逻辑保持一致。

### 8. 失败文案过于尖锐

**现状**：失败文案包含"建议去做产品经理"等嘲讽性内容，可能触及不良价值导向红线。

**建议**：改为中性鼓励语气，例如"还差 {r} 格，再试试吧~"。
